## Задание 1

В качестве результата выполнения первого задания в той же директории, что и отчёт, лежит файл Task1.py
В нём есть 2 функции ```Sum1``` и ```Sum2```, которые отвечают за сложение чисел A и B, и чисел A и B^2 соответственно.
Одна из функций под комментарием так как часть кода с вывовдом в файл поддеживает запись результата одной функции.
Если нужно посмотреть результат второй функции надо убрать комментарий на текущей и поставить его перед работающий сейчас.

## Задание 2

Резцльтат выполнения этого задания находится в файле Task2.py, лежащем в этой же директории. В нём сразу есть часть кода для измерения времени.
Я написал её для задания 4 и оставил, чтобы не загружать отдельно файлы со счётчиком времени и без.

```
Сам алгоритм для подсчёта n-ного числа Фиббаначи работает по принципу
динамического программирования. Мы строим следующее число Фиббаначи
опираясь на значения 2ух предыдущих чисел и, после построения нового числа,
мы меняем значение меньшего из исходных чисел на значение большего,
а само большее исходное на только что построенное.
Таки образом мы всегда поддерживаем только 2 наибольших числа Фиббаначи
и не перегружаем компьютер
```

Описанный выше алгоритм, на мой взгялд, является наиболее быстрым и не нагружающим систему алгоритмом для нахождения
числа Фиббаначи

## Задание 3

Результат выполнения этого задания лежит в той же директории и называется Task3.py. В нём есть часть кода для подсчёта времени,
которую я оставил по тем же причинам, что и в задании 2.

```
Алгоритм работает на идее того, что последняя цифра числа Фиббаначи
повтряется циклично с периодом 60. То есть все числа Фиббаначи
одинаковой кратности имеют в конце одну и ту же цифру.
Поэтому я нахожу кратность числа с помощю команды %, а затем
делаю по аннологии с алгоритмом из задания 2, за исключением того,
что поддерживаю не 2 числа Фиббаначи, а только последние цифры
двух последних чисел Фиббаначи.
```

Получается вообще не нагружающий систему алгоритм.
