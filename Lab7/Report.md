# Отчёт по лабораторной работе №7 (Вариант 24, Задания 2, 5)

## Задание 2

## Описание задания

```
Дан примитивный калькулятор, который может выполнять следующие три опе-
рации с текущим числом x: умножить x на 2, умножить x на 3 или прибавить 1 к x.
Дано положительное целое число n, найдите минимальное количество операций,
необходимых для получения числа n, начиная с числа 1.
• Формат ввода / входного файла (input.txt). Дано одно целое число n,
1 ≤ n ≤ 106. Посчитать минимальное количество операций, необходимых
для получения n из числа 1.

• Формат вывода / выходного файла (output.txt). В первой строке вывести
минимальное число k операций. Во второй – последовательность проме-
жуточных чисел a0, a1, ..., ak−1 таких, что a0 = 1, ak−1 = n и для всех
0 ≤ i < k – ai+1 равно или ai + 1, 2 · ai, или 3 · ai
. Если есть несколько вариантов, выведите любой из них.

• Ограничение по времени. 1 сек.
```

### Пояснение к коду

Алгоритм для решения этого задания построен на методе динамического програмирования.
Основная идея алгоритма - мы бцдем строить список, где для каждого элемента от 0 до n будет 
храниться минимальное количество операций, необходимых, чтобы добраться до него от 1.

Реализация
Создаём список OperationsCounter и заполняем его n+1 нулём. Сначала заполним список с помощью
динамического програмирования. Для этого запустим цикл по всем индексам списка и будем рассматривать 
3 варианта для каждого.

- Если мы попали в текущее число из предыдущего
- Попали из числа, в 2 раза меньшего текцщего значения, если число делится на 2
- И попали из числа в 3 раза меньше текущего значения, если число делится на 3

Из этих трёх вариантов мы выбираем наименьший результат и записываем его в
ячейку списка, индекс котрой равен проверяемому числу. 

В итоге у нас получается список, в котором на каждой позиции стоит число,
соответствующее минимальному количеству операций, необходимых для того, чтобы
попасть в число, равное индексу этой позиции.

Теперь восстановим последовательность чисел. Вводим список Order, в котором
будем хранить весь путь от 1 до необходимого числа. Для того, чтобы его заполнить, начинаем
цикл while, и идём по всем элементам списка OperationConter, проверяя 3 условия каждый раз

- Могло ли текущее число прийти из предыдущего
- Могло ли оно прийтий из числа, в 2 раза меньшего данного
- Могло ли оно прийтий из числа, в 3 раза меньшего данного

В итоге получаем список всех элементов, через которые попали в n из 1 минимальным путём,
добавляем в его конец 1 и разворачиваем, чтобы в выходном файле порядок шёл по возрастанию.
